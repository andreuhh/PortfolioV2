React: 10 most frequently asked questions

Jenuary 2020

web and mobile development

1 : What is React?

React is a Javascript library, created by Facebook / Instagram, which manages the user interface (i.e. the front-end of our applications). It allows us to develop web applications using big data in a simple, fast and scalable way. Many companies such as AirBnB, Netflix, Uber, PayPal use React as a technology.



2 : What is the virtual DOM?

The Virtual DOM is a tree structure of Javascript objects (ReactElements) that represent the current DOM. This representation of the DOM allows, when a change occurs, to re-render the component itself and all the components related to it (This translates into the re-execution of the code for the creation of the Virtual DOM).



3 : What is a componet?

A React application consists of components, each of which can be understood as a graphic component of the application. Each component is independent of the other (self-contained) and can have a state (state) or properties (props). When a state or property changes, React does nothing more than re-render the component.


React rendering is said to be deterministic, because given a defined set of props and states the component will always show the same way. A component in React is defined in JSX, which is a language that allows you to describe HTML in Javascript.



4 : What is a componet?

The properties of a component are called props. The props are the attributes set on the component itself and are used to put the components in communication with each other (parent-child communication).


The properties (props) are immutable, i.e. it is not possible to change them within the child component, this is because the child component is not the owner. A component can access all its properties through the "this.props" object.



5 : What is the state of a component?

The state of a component is called state. The state is used to differentiate the views of a component. Unlike the props, the state is mutable and private to the component because it does not belong to the parent component but rather to the component itself.


When defining the state, it is also necessary to specify the initial value that the state assumes at the time of component initialization. A component accesses its state through the "this.state" object.
